Selection-Based Calculator Program Documentation

This page documents a Python selection-based calculator that asks the user which arithmetic operation to run, then executes only that operation.

Program Purpose

This program improves on a “do-everything” calculator by making execution user-driven. The user selects one operation (e.g., +, -, *, /, or sqrt), and the program performs only that operation.

1) Importing a Library

import math

This line loads Python’s math library, which provides advanced mathematical functions. In this program, it is used to compute a square root: 

math.sqrt(x)

2) Asking the User What They Want to Do

op = input("what operation would you like to perform?, e.g., [+,-,*,/,sqrt] ")

The program asks the user to choose an operation and stores their choice in op. This matters because the program becomes:

  • Interactive (the user controls what happens)
  • Intentional (only the requested operation runs)
  • Less overwhelming (no “information overload”)

3) Using Selection (if / elif / else)

The key concept is selection. The program checks the value of op and decides which code block to execute.

Addition

if op == "+":

If the user types +, the program asks for two numbers and adds them.

Subtraction

elif op == "-":

If the user types -, the program subtracts the second value from the first.

Multiplication

elif op == "*":

Division

elif op == "/":

Square Root (Unary Operation)

elif op == "sqrt":
    x = int(input("please enter value x: "))
    z = math.sqrt(x)
Teaching moment: Not all operations require two numbers.
  • Binary operations need two values: +, -, *, /
  • Unary operations need one value: sqrt

4) Handling Invalid Input

else:
    print("Invalid operation, please try again")

If the user enters something outside the allowed choices, the program does not crash. It handles mistakes gracefully. This is an example of:

  • Defensive programming
  • Anticipating user mistakes
  • Writing safer, more robust code

Subtle but Powerful Idea (Great for Adversarial Thinking)

The program limits what the user can do to only allowed operations: 

[ + , - , * , / , sqrt ]

This is an early example of controlling the input surface to prevent misuse or unexpected behavior— a foundational idea in secure and robust programming.

Python Code (Reference)

import math

op = input("what operation would you like to perform?, e.g., [+,-,*,/,sqrt] ")

if op == "+":
    x = int(input("please enter value x: "))
    y = int(input("please enter value y: "))
    z = x + y
    print("The addition is ", z)

elif op == "-":
    x = int(input("please enter value x: "))
    y = int(input("please enter value y: "))
    z = x - y
    print("The subtraction is ", z)

elif op == "*":
    x = int(input("please enter value x: "))
    y = int(input("please enter value y: "))
    z = x * y
    print("The product is ", z)

elif op == "/":
    x = int(input("please enter value x: "))
    y = int(input("please enter value y: "))
    z = x / y
    print("The division is ", z)

elif op == "sqrt":
    x = int(input("please enter value x: "))
    z = math.sqrt(x)
    print("The sqrt is ", z)

else:
    print("Invalid operation, please try again")
